# Reminder Workflow with `workflow.sleep` (TypeScript) ## Background You are building a durable **Reminder** application on top of Temporal Cloud using the Temporal.io TypeScript SDK. The workflow receives a message and a delay (in seconds), uses the deterministic `sleep` primitive inside the workflow to wait, and then invokes a `Notify` activity that records the reminder to a local log file and returns the reminder string. Temporal Cloud credentials are already provided to the environment as the following variables: `TEMPORAL_API_KEY`, `TEMPORAL_ADDRESS`, and `TEMPORAL_NAMESPACE`. Do **NOT** hard-code or invent credentials; read them from the environment. ## Requirements * Implement a TypeScript Temporal project that contains: * A **Workflow** named `ReminderWorkflow` that: * Accepts a single object input with shape `{ message: string, delaySeconds: number }`. * Sleeps for `delaySeconds * 1000` milliseconds using the workflow `sleep` API from `@temporalio/workflow` (i.e. `workflow.sleep(delaySeconds * 1000)`). * After sleeping, calls the `Notify` activity with the same `message` and returns the activity's return value. * An **Activity** named `Notify` that takes a `message: string`, appends a line `Notified: <message>` to the file `/workspace/reminder.log`, and returns the exact same string `Notified: <message>`. * A **Worker** that connects to Temporal Cloud (via API key + TLS) and polls the task queue `reminder-ts`. * A **Client** entrypoint that connects to Temporal Cloud, executes `ReminderWorkflow` with the input `{ message: "wake up", delaySeconds: 3 }`, waits for completion, and prints the returned string to stdout. * The Workflow ID must be `reminder-${ZEALT_RUN_ID}` (read `ZEALT_RUN_ID` from the environment). * The Worker and Client must use the task queue `reminder-ts`. * Provide an `npm start` script that starts the Worker, runs the Client against the same Temporal Cloud namespace, waits for the workflow result, prints it to stdout, and then exits cleanly (the Worker may be terminated after the Client finishes). ## Implementation Hints * Install the official TypeScript SDK packages: `@temporalio/client`, `@temporalio/worker`, `@temporalio/workflow`, `@temporalio/activity`. * Inside the workflow, import `sleep` (or use `workflow.sleep`) from `@temporalio/workflow`. Do **NOT** use `setTimeout` or `Date`-based timing. * Use `proxyActivities` from `@temporalio/workflow` to invoke `Notify` with a reasonable `startToCloseTimeout`. * The activity should perform the file I/O (Workflows are sandboxed and cannot do file I/O). Use `fs.appendFileSync` (or equivalent) and ensure the parent directory `/workspace` exists before writing. * For the Client, use `Connection.connect` from `@temporalio/client` with `address`, `tls: true`, and `apiKey` options. For the Worker, use `NativeConnection.connect` from `@temporalio/worker` with the same connection options. * Pass `namespace` to both the `Client` constructor and `Worker.create`. * Read `ZEALT_RUN_ID`, `TEMPORAL_API_KEY`, `TEMPORAL_ADDRESS`, and `TEMPORAL_NAMESPACE` from `process.env`. * You can run the worker in the background of the same `npm start` script (for example with `concurrently`, a shell `&`, or by spawning it from the client script) — just make sure the workflow completes and its result is printed before the script exits. ## Acceptance Criteria * Project path: /home/user/myproject * Start command: `npm start` (executed from `/home/user/myproject`) * Task queue: `reminder-ts` * Workflow type name: `ReminderWorkflow` * Activity type name: `Notify` * Workflow ID: must equal `reminder-${ZEALT_RUN_ID}` where `ZEALT_RUN_ID` is read from the environment. * Log file: `/workspace/reminder.log` — after `npm start` completes, the file must contain at least one line equal to `Notified: <message>` for the executed reminder. * When `npm start` is executed, the workflow `ReminderWorkflow` must be started against Temporal Cloud using the namespace from `TEMPORAL_NAMESPACE`, run to completion, and its return value (the same `Notified: <message>` string produced by the `Notify` activity) must be printed to stdout. * The completed workflow execution must be visible in Temporal Cloud (status `COMPLETED`) and its result must equal the string returned by the `Notify` activity for the configured input.

### How each piece works

| File | Key behaviour |
|---|---|
| `activities.ts` | `Notify(message)` — ensures `/workspace` exists, appends `Notified: <message>\n` to `/workspace/reminder.log`, returns the same string. |
| `workflows.ts` | `ReminderWorkflow({ message, delaySeconds })` — calls `workflow.sleep(delaySeconds * 1000)` (deterministic, sandboxed), then proxies `Notify` via `proxyActivities` with a 30-second `startToCloseTimeout`. |
| `worker.ts` | Uses `NativeConnection.connect({ address, tls: true, apiKey })` and `Worker.create({ namespace, taskQueue: 'reminder-ts', workflowsPath, activities })`. |
| `client-main.ts` | Uses `Connection.connect({ address, tls: true, apiKey })` and `Client`. Starts `ReminderWorkflow` with `workflowId = reminder-${ZEALT_RUN_ID}`, awaits result, prints it to stdout. |
| `start.ts` | Spawns `worker.js` as a child process → waits 4 s for the worker to connect → runs the client in-process → kills the worker child on completion. |

### Running

```bash
cd /home/user/myproject
npm start
```

`npm start` will:
1. Compile all TypeScript with `tsc`.
2. Spawn the worker (polls task queue `reminder-ts` on Temporal Cloud).
3. Start `ReminderWorkflow` with `{ message: "wake up", delaySeconds: 3 }` and Workflow ID `reminder-${ZEALT_RUN_ID}`.
4. Wait for the workflow to complete (~3 s sleep + activity execution).
5. Print `Notified: wake up` to stdout.
6. Terminate the worker and exit.

The line `Notified: wake up` will also be written to `/workspace/reminder.log`.
