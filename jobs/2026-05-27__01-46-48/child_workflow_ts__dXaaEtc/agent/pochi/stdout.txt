### Key design decisions

| Concern | Approach |
|---|---|
| `workflowsPath` | Points to `src/workflows.ts` (TypeScript source); Temporal's bundler (webpack) compiles it at worker startup — required because the bundler does not accept pre-compiled `.js` files |
| Child workflow IDs | `child-double-{index}-{n}` — unique per element, collision-safe within a single parent run |
| Graceful shutdown | `run.ts` spawns the worker, waits 5 s for it to reach `RUNNING`, runs the client to completion, then sends `SIGTERM`; the worker drains cleanly |
| Credentials | All read from `TEMPORAL_API_KEY`, `TEMPORAL_ADDRESS`, `TEMPORAL_NAMESPACE`, `ZEALT_RUN_ID` — never hard-coded |
