### Key design decisions

**`workflow.ts`**
- `defineUpdate<number, [number]>('deposit')` — return type `number`, single `amount: number` arg.
- `setHandler(depositUpdate, handler, { validator })` — the non-async validator throws `ApplicationFailure.create({ type: 'InvalidAmount' })` for `amount <= 0`, rejecting the update without mutating state or recording a `WorkflowExecutionUpdateAccepted` event.
- `defineSignal('finish')` + `condition(() => finished)` — no busy-loops, no non-deterministic APIs.
- Returns `balance` (the number `175`) as the workflow result.

**`worker.ts`**
- Uses `NativeConnection.connect({ address, tls: true, apiKey })` for Temporal Cloud.
- `workflowsPath` points to `src/workflow.ts` so the Temporal bundler can webpack it from TypeScript source.

**`client.ts`**
- Uses `Connection.connect({ address, tls: true, apiKey })` + `new Client({ connection, namespace })`.
- Starts the workflow with `workflowIdConflictPolicy: 'TERMINATE_EXISTING'` for safe re-runs.
- Calls `handle.executeUpdate(depositUpdate, { args: [amount] })` for each of the three deposits and prints `Updated balance: <n>`.
- Signals `finishSignal` then awaits `handle.result()` and prints `Final balance: <n>`.

**`start.ts`**
- Spawns the worker process in the background, pauses 3 s for it to connect, runs the client process, then `SIGTERM`s the worker after the client exits cleanly.
