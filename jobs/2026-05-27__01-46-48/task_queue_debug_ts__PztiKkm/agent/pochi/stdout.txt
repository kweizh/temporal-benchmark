## Root Cause

The Task Queue names were mismatched:

| File | Task Queue |
|---|---|
| `src/worker.ts` | `task-queue-B` |
| `src/client.ts` | `task-queue-A` |

A Temporal Worker only polls the single Task Queue it is registered for. The Client was scheduling `PingWorkflow` on `task-queue-A`, but the Worker was listening on `task-queue-B` — so no Worker ever picked up the Workflow Task, leaving the execution stuck in `Running` with a scheduled-but-never-started Workflow Task indefinitely.

## Fix Applied

One-line change in `src/worker.ts`:

```diff
-    taskQueue: 'task-queue-B',
+    taskQueue: 'task-queue-A',
```

Both the Worker and the Client now agree on `task-queue-A`.
