# Money Transfer Saga with Compensation (TypeScript) ## Background You are building a durable **Money Transfer** application on top of Temporal Cloud using the Temporal.io TypeScript SDK. Transfers are modeled as a Saga: the workflow executes a `withdraw` activity followed by a `deposit` activity. If `deposit` fails, the workflow **MUST** compensate by calling a `refund` activity to restore the originating account balance, and then surface the failure so the workflow execution finishes in a `FAILED` terminal state. All balance state is persisted in a single JSON file at `/workspace/accounts.json`. This file is pre-initialized in the environment to `{ "A": 100, "B": 0 }` and is the only source of truth for account balances. The verifier reads this file directly to confirm that compensation happened correctly.
