# Subscription Workflow with Signals & Queries (TypeScript) ## Background You are building a **Subscription** service on top of Temporal Cloud using the Temporal.io TypeScript SDK. The workflow runs a recurring billing cycle and must accept external requests during the run: a user may **upgrade** their tier or **cancel** their subscription at any moment. The workflow also exposes its current state via a **query** so a client can inspect it without waiting for the workflow to finish. This pattern combines three of Temporal's message-passing primitives — Signals (to mutate workflow state from outside), Queries (to read workflow state from outside), and durable Timers (`workflow.sleep`) — together with an Activity that performs side effects (writing to a log file). Temporal Cloud credentials are already provided to the environment as the following variables: `TEMPORAL_API_KEY`, `TEMPORAL_ADDRESS`, and `TEMPORAL_NAMESPACE`. Do **NOT** hard-code or invent credentials; read them from the environment. The current run id is provided as `ZEALT_RUN_ID`; use it to scope the workflow id. ## Requirements Implement a TypeScript Temporal project that contains: * A **Workflow** named `SubscriptionWorkflow` that: * Accepts a single object input with shape `{ userId: string, tier: string }`. * Iterates through up to **12** billing periods. Between periods it must wait by calling the workflow `sleep` API with `2000` ms (`workflow.sleep(2000)`). * In each period (and before exiting), it must invoke an Activity `chargeCard(userId, tier)` with the **current** tier (which may have been changed by an `upgrade` signal). * Defines a Signal named `upgrade` that accepts a single `newTier: string` argument and updates the current tier going forward. * Defines a Signal named `cancel` (no arguments) that causes the workflow to exit cleanly at the next opportunity. After receiving `cancel`, the workflow must not start any new billing period. * Defines a Query named `getStatus` that returns the current state as `{ tier: string, billings: number, cancelled: boolean }` (where `billings` is the number of completed `chargeCard` calls so far). * Returns a final object of shape `{ billings: number, finalTier: string, cancelled: boolean }` from the workflow. `billings` is the total number of completed `chargeCard` invocations; `finalTier` is the tier that was in effect at the end of the run; `cancelled` is `true` iff the `cancel` signal caused the early exit. * An **Activity** named `chargeCard` that takes `(userId: string, tier: string)` and appends a single line `<userId>:<tier>` (no extra spaces, terminated with a newline) to the file `/workspace/charges.log`. The activity must ensure `/workspace` exists before writing. * A **Worker** that connects to Temporal Cloud (API key + TLS) and polls the task queue `sub-ts`. * A **Client** entrypoint that, against the same Temporal Cloud namespace: 1. Starts `SubscriptionWorkflow` with input `{ userId: "u1", tier: "basic" }`, task queue `sub-ts`, and workflow id `sub-${ZEALT_RUN_ID}`. 2. After waiting **1 second** of real time, sends the `upgrade` signal with argument `"premium"`. 3. After waiting another **2 seconds** of real time (so \~3 seconds since start), calls the `getStatus` query and logs the returned object to stdout. 4. After waiting another **2 seconds** of real time (so \~5 seconds since start), sends the `cancel` signal. 5. Awaits the workflow's final result and prints it to stdout as `Final result: <json>` where `<json>` is `JSON.stringify(result)`. The project must expose an `npm start` script that starts the Worker, runs the Client, waits for the workflow to complete, prints the final result, and then exits cleanly with exit code `0`. ## Implementation Hints * Install the official TypeScript SDK packages: `@temporalio/client`, `@temporalio/worker`, `@temporalio/workflow`, `@temporalio/activity`. * Inside the workflow, use `defineSignal`, `defineQuery`, `setHandler`, `condition`, and `sleep` from `@temporalio/workflow`. Do **NOT** use `setTimeout`, `Date`, or any non-deterministic API inside the workflow function. * Use `proxyActivities` from `@temporalio/workflow` to invoke `chargeCard` with a reasonable `startToCloseTimeout`. * Activities perform file I/O (workflows are sandboxed). In `chargeCard`, ensure `/workspace` exists and append `"<userId>:<tier>\n"` to `/workspace/charges.log`. * The client should use `Connection.connect` from `@temporalio/client` with `address`, `tls: true`, and `apiKey`. The worker should use `NativeConnection.connect` from `@temporalio/worker` with the same connection options. Pass `namespace` to both the `Client` constructor and `Worker.create`. * Read `ZEALT_RUN_ID`, `TEMPORAL_API_KEY`, `TEMPORAL_ADDRESS`, and `TEMPORAL_NAMESPACE` from `process.env`. * Wait for incoming signals using `workflow.condition(...)` rather than busy-looping. The cancel signal should make the workflow break out of its billing loop without throwing — it must finish in `COMPLETED` status, **not** `CANCELED` or `FAILED`. * You may run the worker in the background of the same `npm start` script (with a shell `&`, `concurrently`, or by spawning it from the client script).
