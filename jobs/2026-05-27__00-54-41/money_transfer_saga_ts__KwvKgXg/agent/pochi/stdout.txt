# Money Transfer Saga with Compensation (TypeScript) ## Background You are building a durable **Money Transfer** application on top of Temporal Cloud using the Temporal.io TypeScript SDK. Transfers are modeled as a Saga: the workflow executes a `withdraw` activity followed by a `deposit` activity. If `deposit` fails, the workflow **MUST** compensate by calling a `refund` activity to restore the originating account balance, and then surface the failure so the workflow execution finishes in a `FAILED` terminal state. All balance state is persisted in a single JSON file at `/workspace/accounts.json`. This file is pre-initialized in the environment to `{ "A": 100, "B": 0 }` and is the only source of truth for account balances. The verifier reads this file directly to confirm that compensation happened correctly. Temporal Cloud credentials are already provided to the environment as the following variables: `TEMPORAL_API_KEY`, `TEMPORAL_ADDRESS`, and `TEMPORAL_NAMESPACE`. Do **NOT** hard-code or invent credentials; read them from the environment. ## Requirements * Implement a TypeScript Temporal project that contains: * A **Workflow** named `MoneyTransfer` that: * Accepts a single object input with shape `{ fromAccount: string, toAccount: string, amount: number }`. * Sequentially calls activity `withdraw(fromAccount, amount)`, then activity `deposit(toAccount, amount)`. * If `deposit` throws / fails (after Temporal's retry policy is exhausted), the workflow **MUST** execute a compensating activity `refund(fromAccount, amount)` and then propagate the original failure (either re-throw, or fail the workflow execution in any way that produces a `FAILED` terminal status in Temporal Cloud). * On the happy path (no error), the workflow finishes normally with a `COMPLETED` terminal status. * **Activities** that all read/write `/workspace/accounts.json` atomically (read full file → mutate → write full file): * `withdraw(account: string, amount: number)`: subtract `amount` from `accounts[account]`. * `deposit(account: string, amount: number)`: add `amount` to `accounts[account]`. **If `account === "B_FAIL"`, throw an error so the activity fails.** This failure must be classified so that Temporal does not retry it forever — limit total attempts to a small number (e.g. `maximumAttempts: 1` or `2`) so the saga finishes within a reasonable time. * `refund(account: string, amount: number)`: add `amount` to `accounts[account]` (i.e., reverse a prior `withdraw`). * A **Worker** that connects to Temporal Cloud (via API key + TLS) and polls the task queue `saga-ts`. * A **Client** entrypoint that runs **TWO** `MoneyTransfer` workflow executions sequentially against Temporal Cloud, **from a single client script**: 1.
